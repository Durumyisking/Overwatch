# Overwatch Current Milestone 01

## 결론
현재 작업 목표는 아래 한 줄로 고정한다.

`2인 멀티플레이 전용 서버에서, 영웅 1명이 스폰되고, 기본 공격으로 피해를 주고, 죽고, 리스폰하고, HUD가 이를 표시하는 상태`

이 마일스톤은 이후 GAS, CommonUI, MVVM 확장의 기준선이다.

---

## 이번 마일스톤의 범위

포함:
- 전용 서버 실행
- 클라이언트 2명 접속
- 영웅 1명 스폰
- 이동
- 기본 공격
- 피해 적용
- 사망 처리
- 리스폰 처리
- HUD 체력 표시

제외:

- 궁극기
- 점수판
- 매치 승패

## 구조와 책임

## GameMode
책임:
- 서버 전용 규칙
- 플레이어 입장 처리
- 초기 스폰
- 사망 후 리스폰 시점 결정

하지 말 것:
- UI 값 저장
- 클라이언트 전용 표시 로직 처리

## GameState
책임:
- 매치 공용 상태가 필요해질 때 그 값을 복제

현재 단계:
- 필요 최소 상태만 유지
- 억지로 많은 값을 두지 않는다

## PlayerState
책임:
- 플레이어 지속 데이터 owner
- 이후 ASC owner 후보
- 처치/죽음/선택 영웅/궁극기 게이지 같은 지속 데이터의 자리

현재 단계:
- 최소한의 플레이어 지속 상태 자리 확보

## Character
책임:
- 현재 아바타
- 이동
- 피격 반응
- 현재 체력 표시를 위한 소스 연결
- 사망 상태 전환

주의:
- respawn 이후 유지되어야 하는 데이터를 모두 Character에 고정하지 않는다

## PlayerController
책임:
- 입력 처리
- 로컬 HUD 생성 요청
- 서버 RPC 시작점

## HUD / Widget / ViewModel
책임:
- 체력 표시

주의:
- 체력을 소유하거나 수정하지 않는다
- authoritative owner의 값을 읽기 좋은 형태로만 보여준다

---

## 이번 작업의 세부 구현 리스트

1. 전용 서버와 클라이언트 2인 접속 루프 확보
2. 플레이어 스폰 경로 확인
3. 영웅 1명 베이스 캐릭터 준비
4. 기본 공격 입력 -> 서버 요청 경로 연결
5. 서버에서 피해 계산 및 적용
6. 체력이 0 이하일 때 사망 상태 전환
7. 일정 시간 후 리스폰
8. HUD에서 현재 체력 표시

---

## 데이터 흐름 기준

### 공격
`입력 -> PlayerController -> Server RPC -> Character/Combat owner -> 피해 적용`

### 체력 표시
`Authoritative state -> ViewModel 또는 HUD projection -> Widget 표시`

### 사망 / 리스폰
`피해 누적 -> 체력 0 -> 서버 사망 확정 -> GameMode가 리스폰 orchestration`

---

## 완료 조건
- 전용 서버에서 클라이언트 2명이 접속된다.
- 각 클라이언트는 같은 영웅으로 스폰된다.
- 기본 공격이 서버 기준으로만 피해를 확정한다.
- 피해 결과가 다른 클라이언트에도 일관되게 보인다.
- 체력이 0 이하가 되면 사망한다.
- 사망 후 다시 리스폰된다.
- HUD가 현재 체력을 표시한다.

---

## 검증 체크리스트
- 두 클라이언트 모두 접속 가능한가
- 서로를 공격할 수 있는가
- 피해량이 서버 기준으로만 적용되는가
- 사망 시 중복 처리나 유령 상태가 없는가
- 리스폰 후 조작이 정상 복구되는가
- HUD 체력 값이 실제 상태와 일치하는가

## 작업 규칙
- 먼저 현재 owner와 authority를 확인하고 수정한다.
- 한 번에 크게 바꾸지 말고 compile-safe 패치로 자른다.
- 완료 판정은 빌드 또는 실행 검증으로 한다.
- 구조가 흔들리면 기능 추가보다 책임 재정의를 먼저 한다.
