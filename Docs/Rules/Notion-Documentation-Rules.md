# Notion Documentation Rules

## General
- Update existing pages/databases instead of creating duplicates.
- Write all documentation in Korean.
- Use commit SHA when available.

## Record Types
- Progress logs: one page per commit.
- Decision logs: meaningful design or policy decisions only.
- Risk logs: active blockers, regressions, or unresolved concerns only.

## Fixed Names
- 프로젝트 개요
- 진행 기록 DB
- 결정 기록 DB
- 리스크 기록 DB
- 주간 요약 DB

## Title Format
- Weekly summary title format: `YYYY.MM.DD 주간 요약`
